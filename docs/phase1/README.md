# Phase 1 — Shipped MVP specifications

This folder holds the canonical, **current** specification for the AstroSocial MVP that
shipped in milestones M1–M12 plus the three-column app-shell redesign.

- `product-requirements.md` — PRD
- `functional-design.md` — functional design
- `architecture.md` — technical specification (authoritative DB schema)
- `repository-structure.md` — directory layout & conventions
- `development-guidelines.md` — **shared** coding standards (apply to all phases)
- `design-guidelines.md` — **shared** UI/UX + app-shell rules (apply to all phases)
- `glossary.md` — ubiquitous language
- `implementation-plan.md` — the M0–M12 roadmap (all merged)
- `ideas/` — original brainstorming inputs

> `development-guidelines.md` and `design-guidelines.md` are cross-cutting and remain the
> source of truth for Phase 2 work too. The `/add-feature` workflow reads
> `docs/phase1/design-guidelines.md`.
